这个站点的目标不是做一个短期展示页，而是做一个能长期陪我积累知识的系统。

核心原则很简单：

1. 生产环境尽量静态化，部署成本接近 0。
2. 内容保存在 Git 和 MDX 里，不被任何数据库或平台锁死。
3. AI 只在写作和构建阶段重度参与，访问者打开页面时不消耗任何 API。

## 架构

```text
drafts/*.mdx
  -> scripts/ai-processor.py
  -> src/content/*
  -> Astro build
  -> Cloudflare Pages / Vercel
```

这意味着每一篇文章都可以在发布前经过自动审稿、SEO 生成、标签分类和多语言处理。

## 为什么不是动态博客

动态博客当然可以做得很强，但个人长期使用时，维护成本会变成隐形负担。静态站的优势是：

- 没有数据库迁移。
- 没有服务器账单。
- 页面速度快。
- 备份和迁移简单。

长期来看，简单可靠比一开始功能爆炸更重要。
